What's Fibonacci Sequence?

1, 1, 2, 3, 5, 8, 13, 21...

Basically, the Fibonacci Sequence is a sequence that starts with two 1, and all the following numbers are the sum of the previous two numbers.

1, 1, 2, 3

For example, the fourth number 3, is the sum of the previous two numbers 1 and 2.

1, 1, 2, 3, 5

The fifth number 5, on the sequence is the sum of the previous two numbers 2 and 3.

So now let's start with our question.

Please find the n-th number on the Fibonacci Sequence.

Let's define a function fib(n) which will accept one parameter n. Please complete the function to display the Fibonacci sequence up to the nth term.

    function fib(n) {}

Example input:

67

Expected output:

813

Since the n-th term in the sequence is always the sum of the previous two numbers. So we can do it with a straightforward Recursive Solution:

                            if(n === 1 || n === 2) {
                                return 1;
                            }
                            else {
                                return fib(n - 1) + fib(n - 2)
                            }

You may find this solution works but actually, it's very very inefficient. Let's see why.

Here is an example we try to find the 6th Fibonacci number. To do that, we need to find out the 5th and 4th Fibonacci numbers and add them together. And to find the 5th Fibonacci number, we need to find out the 4th and 3rd Fibonacci numbers and add them together. And as you can see, the problem with this approach is that there are a lot of duplications. For instance, in order to get the 3rd Fibonacci number, we repeat the competition three times.

And we repeated twice in order to get the 4th Fibonacci number.

This is very slow and inefficient. And the time to find the n-th Fibonacci number will be O(2^n).

Dynamic Programming with memoization Solution

In order to solve the problem, we can cache all the intermediate results.

You could either do it using the bottom-up solution or map(). Here are the examples for both of them.

Bottom-Up

Maybe you would like to think of your own solution first, click here when you done 
                        
                        const fib = n => {
                            const memo = [];
                            for (let i = 0; i <= n; i++) {
                                if (i < 2)   memo[i] = i;
                                else memo[i] = memo[i-2] + memo[i-1];
                            }
                            return memo[n];
                        }

So for this solution, we created an empty array, and iterate from 0 to n. For instance, we still want to find the 6th Fibonacci number. As we run the code above. When i < 2, we just put 'i' into the array. [0, 1] When i >= 2, We will try to find the 6th number, and it will search all the way from the 6th number to the 2nd number. Just like the image below indicated.

As we did the search, we are going to collect the results. when it comes to the 2nd Fibonacci number, it's 1. And then the 3rd number should be 2, and the 5th number will go 3 and so on.

In this solution, we just created one array, and store all the results in between. And the time complexity will go down to O(n) since we don't do any duplicated competition anymore.

Map()

                            const fib = (n, memo) => {
                                memo = memo || new Map();

                                if (memo.has(n)) return memo.get(n);
                                if (n < 2) return n;

                                memo.set(n, fib(n - 1, memo) + fib(n - 2, memo));
                                return memo.get(n)
                            }

As a matter of fact, we only need the previous two values instead of the store all the values in one array or one Map.

Please try to only use the previous two values to get the n-th term Fibonacci number.
